# Fader

`Fader` is a dB control for console-style channel strips. It defaults to vertical orientation, `-60..+12 dB`, `0.1 dB` steps, a `0 dB` unity point, and a non-linear scale with more usable resolution around common mix positions.

<DemoWithCode source={FaderDemoSource} cssSource={FaderDemoStylesSource}>
  <FaderDemo />
</DemoWithCode>

## Anatomy

```tsx
import { Fader } from "@ryhrm-gz/audio-ui-react";

<Fader.Root>
  <Fader.Scale />
  <Fader.Track>
    <Fader.Range />
    <Fader.Thumb />
  </Fader.Track>
  <Fader.Value />
  <Fader.HiddenInput />
</Fader.Root>;
```

## Scale marks

`Fader.Scale` renders the fader law marks from the core state. You can pass a custom `scale` array to `Fader.Root` when a product needs a different response curve.

```tsx
<Fader.Root
  defaultValue={0}
  min={-80}
  max={10}
  unity={0}
  scale={[
    { value: -80, percent: 0, label: "-inf" },
    { value: -40, percent: 0.2, label: "-40" },
    { value: -10, percent: 0.55, label: "-10" },
    { value: 0, percent: 0.75, label: "0" },
    { value: 10, percent: 1, label: "+10" },
  ]}
>
  {/* fader parts */}
</Fader.Root>
```

## Horizontal faders

Pass `orientation="horizontal"` to map pointer movement from left to right while preserving the same fader law.

```tsx
<Fader.Root defaultValue={-6} orientation="horizontal">
  <Fader.Scale />
  <Fader.Track>
    <Fader.Range />
    <Fader.Thumb aria-label="Channel gain" />
  </Fader.Track>
</Fader.Root>
```

## Styling hooks

Fader parts expose `--fader-percent`, `--fader-unity-percent`, and `--fader-mark-percent`.

```css
.fader-thumb {
  top: calc((1 - var(--fader-percent)) * 100%);
}

.fader-scale [data-part="scale-mark"] {
  top: calc((1 - var(--fader-mark-percent)) * 100%);
}

.fader-thumb[data-orientation="horizontal"] {
  left: calc(var(--fader-percent) * 100%);
  top: 50%;
}

.fader-scale[data-orientation="horizontal"] [data-part="scale-mark"] {
  left: calc(var(--fader-mark-percent) * 100%);
  top: auto;
}
```

## API reference

### `Fader.Root`

`Fader.Root` renders a `div`, owns fader state, and provides context to the other parts. It accepts standard `div` props except `defaultValue`, `onChange`, and `children`, which are replaced by the fader-specific API.

| Prop                 | Type                                   | Default          | Description                                                                                                                                                    |
| -------------------- | -------------------------------------- | ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `value`              | `number`                               | -                | Controlled value in the fader range. Pass with `onValueChange` when external state owns the fader value.                                                       |
| `defaultValue`       | `number`                               | `min` or `0`     | Initial uncontrolled value. Also becomes the double-click reset target.                                                                                        |
| `min`                | `number`                               | `-60`            | Lower dB bound for the fader. Values and scale points are clamped into the resolved range.                                                                     |
| `max`                | `number`                               | `12`             | Upper dB bound for the fader. If `min` is greater than `max`, the core range resolver normalizes the order.                                                    |
| `step`               | `number`                               | `0.1`            | Value increment for pointer and keyboard changes. Non-positive or invalid steps fall back to the default.                                                      |
| `orientation`        | `"horizontal" \| "vertical"`           | `"vertical"`     | Controls whether pointer movement maps left-to-right or bottom-to-top.                                                                                         |
| `unity`              | `number`                               | `0`              | Unity gain value. The state exposes `unityPercent`, and the root adds `data-unity` when the current value equals unity.                                        |
| `scale`              | `readonly FaderScalePoint[]`           | console dB curve | Maps values to physical throw positions. Each point is `{ value, percent, label? }`, where `percent` is clamped from `0` to `1`; missing labels are formatted. |
| `inverted`           | `boolean`                              | `false`          | Reverses the value-to-position mapping and adds `data-inverted`.                                                                                               |
| `disabled`           | `boolean`                              | `false`          | Prevents pointer and keyboard interaction, removes the thumb from tab order, and adds disabled state attributes.                                               |
| `readOnly`           | `boolean`                              | `false`          | Keeps the thumb focusable while preventing value changes, and adds readonly state attributes.                                                                  |
| `fineControl`        | `boolean \| number`                    | `true`           | When `true`, Shift scales pointer drag movement to 10% for fine adjustment. Pass `false` to disable, or a number to set the drag factor.                       |
| `resetOnDoubleClick` | `boolean`                              | `true`           | Allows a double click on the track to reset to `defaultValue`, or to `min`/`0` when no default value is supplied.                                              |
| `allowTrackClick`    | `boolean`                              | `false`          | When true, pointer down on the track jumps to the pointer position before dragging. When false, dragging starts from the thumb only.                           |
| `name`               | `string`                               | -                | Default form field name consumed by `Fader.HiddenInput`.                                                                                                       |
| `required`           | `boolean`                              | -                | Default required flag consumed by `Fader.HiddenInput`.                                                                                                         |
| `children`           | `ReactNode \| (state) => ReactNode`    | -                | Parts to render. Function children receive the render state, including fader state plus interaction flags.                                                     |
| `onValueChange`      | `(value: number) => void`              | -                | Fires when interaction produces a new clamped and snapped value.                                                                                               |
| `onValueCommit`      | `(value: number) => void`              | -                | Fires when an interaction commits, such as pointer up, keyboard step, or reset.                                                                                |
| `render`             | `RenderProp<ElementProps, FaderState>` | -                | Replaces the root element while preserving generated props and render state.                                                                                   |

### Parts

| Part                | Props and behavior                                                                                                                                                                                                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Fader.Scale`       | Renders a `span` containing marks from the resolved fader scale. It accepts standard `span` props except `children`, `defaultValue`, and `onChange`, plus `children` and `render`. Function children receive `(mark, state)` for custom mark labels. Each mark exposes `--fader-mark-percent`. |
| `Fader.Track`       | Renders a `div` and accepts standard `div` props plus `render`. It owns pointer drag handling, optional track click handling, and double-click reset. It exposes `data-part="track"`, orientation state, inverted/dragging state, `--fader-percent`, and `--fader-unity-percent`.              |
| `Fader.Range`       | Renders an `aria-hidden` `span` and accepts standard `span` props plus `render`. Use it for the visual fill driven by the current fader position; it exposes `--fader-percent` and `--fader-unity-percent`.                                                                                    |
| `Fader.Thumb`       | Renders a focusable `span` with `role="slider"` and generated ARIA value metadata. It accepts standard `span` props except `defaultValue` and `onChange`, plus `render`. Arrow keys, Page Up/Down, Home, and End are handled by the generated keyboard behavior.                               |
| `Fader.Value`       | Renders a `span` connected to the thumb by `aria-describedby`. It accepts standard `span` props except `children`, plus `children`, `format`, and `render`. `format(value, state)` customizes display text when explicit children are not supplied.                                            |
| `Fader.HiddenInput` | Renders `<input type="hidden">` for native form submission. It accepts input props except `children`, `type`, and `value`; `name`, `required`, and `disabled` default to the matching root props and can be overridden directly on the hidden input.                                           |
