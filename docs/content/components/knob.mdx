# Knob

`Knob` is a rotary control for gain, pan, filter cutoff, send amount, and similar audio parameters. It uses vertical pointer dragging and keyboard interaction while exposing `--knob-angle` for styling.

<KnobDemo />

## Anatomy

```tsx
import { Knob } from "@ryhrm-gz/audio-ui-react";

export function GainKnob() {
  return (
    <Knob.Root defaultValue={-6} min={-60} max={12} step={1} name="gain">
      <Knob.Control aria-label="Gain">
        <Knob.Thumb />
      </Knob.Control>
      <Knob.Value format={(value) => `${value} dB`} />
      <Knob.HiddenInput />
    </Knob.Root>
  );
}
```

## Styling hooks

Use `--knob-angle` to rotate the visual thumb and state attributes to branch on interaction state.

```css
.knob-thumb {
  transform: rotate(var(--knob-angle));
}

[data-audio-ui="knob"][data-disabled] {
  opacity: 0.5;
}
```

## API reference

### `Knob.Root`

`Knob.Root` renders a `div`, owns knob state, and provides context to the other parts. It accepts standard `div` props except `defaultValue`, `onChange`, and `children`, which are replaced by the knob-specific API.

| Prop                 | Type                                  | Default      | Description                                                                                                                              |
| -------------------- | ------------------------------------- | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `value`              | `number`                              | -            | Controlled value. Pass with `onValueChange` when external state owns the knob value.                                                     |
| `defaultValue`       | `number`                              | `min` or `0` | Initial uncontrolled value. Also becomes the double-click reset target.                                                                  |
| `min`                | `number`                              | `0`          | Lower value bound. Values are clamped into the resolved range.                                                                           |
| `max`                | `number`                              | `100`        | Upper value bound. If `min` is greater than `max`, the core range resolver normalizes the order.                                         |
| `step`               | `number`                              | `1`          | Value increment for pointer and keyboard changes. Non-positive or invalid steps fall back to the default.                                |
| `minAngle`           | `number`                              | `-135`       | Angle, in degrees, used for the minimum value and exposed through `--knob-angle`.                                                        |
| `maxAngle`           | `number`                              | `135`        | Angle, in degrees, used for the maximum value and exposed through `--knob-angle`.                                                        |
| `disabled`           | `boolean`                             | `false`      | Prevents pointer and keyboard interaction, removes the control from tab order, and adds disabled state attributes.                       |
| `readOnly`           | `boolean`                             | `false`      | Keeps the control focusable while preventing value changes, and adds readonly state attributes.                                          |
| `fineControl`        | `boolean \| number`                   | `true`       | When `true`, Shift scales pointer drag movement to 10% for fine adjustment. Pass `false` to disable, or a number to set the drag factor. |
| `resetOnDoubleClick` | `boolean`                             | `true`       | Allows a double click on the control to reset to `defaultValue`, or to `min`/`0` when no default value is supplied.                      |
| `name`               | `string`                              | -            | Default form field name consumed by `Knob.HiddenInput`.                                                                                  |
| `required`           | `boolean`                             | -            | Default required flag consumed by `Knob.HiddenInput`.                                                                                    |
| `children`           | `ReactNode \| (state) => ReactNode`   | -            | Parts to render. Function children receive the render state, including knob state plus interaction flags.                                |
| `onValueChange`      | `(value: number) => void`             | -            | Fires when interaction produces a new clamped and snapped value.                                                                         |
| `onValueCommit`      | `(value: number) => void`             | -            | Fires when an interaction commits, such as pointer up, keyboard step, or reset.                                                          |
| `render`             | `RenderProp<ElementProps, KnobState>` | -            | Replaces the root element while preserving generated props and render state.                                                             |

### Parts

| Part               | Props and behavior                                                                                                                                                                                                                                                                               |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Knob.Control`     | Renders a focusable `div` with `role="slider"` and generated ARIA value metadata. It accepts standard `div` props except `defaultValue` and `onChange`, plus `render`. Vertical pointer dragging, double-click reset, Arrow keys, Page Up/Down, Home, and End are handled by generated behavior. |
| `Knob.Thumb`       | Renders an `aria-hidden` `span` and accepts standard `span` props plus `render`. Use it for the visual indicator; it exposes `--knob-value`, `--knob-percent`, and `--knob-angle` for styling.                                                                                                   |
| `Knob.Value`       | Renders a `span` connected to the control by `aria-describedby`. It accepts standard `span` props except `children`, plus `children`, `format`, and `render`. `format(value, state)` customizes display text when explicit children are not supplied.                                            |
| `Knob.HiddenInput` | Renders `<input type="hidden">` for native form submission. It accepts input props except `children`, `type`, and `value`; `name`, `required`, and `disabled` default to the matching root props and can be overridden directly on the hidden input.                                             |
