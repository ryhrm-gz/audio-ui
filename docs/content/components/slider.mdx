# Slider

`Slider` is a linear control for normalized parameters, pan, wet/dry balance, bounded ranges, and other values that map cleanly to a percentage. Pass a `number[]` with one or more thumb values.

## Anatomy

```tsx
import { Slider } from "@ryhrm-gz/audio-ui-react";

<Slider.Root defaultValue={[50]}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb index={0} />
  </Slider.Track>
  <Slider.Value />
  <Slider.HiddenInput />
</Slider.Root>;
```

Use additional values for multi-thumb selection.

```tsx
<Slider.Root defaultValue={[20, 80]}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb index={0} aria-label="Minimum" />
    <Slider.Thumb index={1} aria-label="Maximum" />
  </Slider.Track>
  <Slider.Value />
  <Slider.HiddenInput />
</Slider.Root>
```

Set thumb-specific bounds with the `thumbs` array.

```tsx
<Slider.Root defaultValue={[20, 50, 80]} thumbs={[{ max: 35 }, { min: 35, max: 65 }, { min: 65 }]}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb index={0} aria-label="Low" />
    <Slider.Thumb index={1} aria-label="Mid" />
    <Slider.Thumb index={2} aria-label="High" />
  </Slider.Track>
</Slider.Root>
```

### Range

`defaultValue={[lower, upper]}` enables two-thumb behavior. Any number of values can be supplied; values are normalized in ascending order and each thumb receives bounded ARIA metadata.

### Thumb constraints

Use `minStepsBetweenThumbs` to keep thumbs at least a number of steps apart. `minDistance` is also available when a value-unit distance is more convenient. Use `thumbs` for per-thumb `min` and `max` bounds.

```tsx
<Slider.Root defaultValue={[20, 80]} step={5} minStepsBetweenThumbs={2}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb index={0} aria-label="Minimum" />
    <Slider.Thumb index={1} aria-label="Maximum" />
  </Slider.Track>
</Slider.Root>
```

```tsx
<Slider.Root defaultValue={[20, 50, 80]} thumbs={[{ max: 30 }, { min: 40, max: 60 }, { min: 70 }]}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb index={0} aria-label="Low" />
    <Slider.Thumb index={1} aria-label="Mid" />
    <Slider.Thumb index={2} aria-label="High" />
  </Slider.Track>
</Slider.Root>

```

## Range origin

Use `origin` to choose where the visual range starts. The default is `left`; `center` works well for pan controls, and `right` fills back from the end of the track.

```tsx
<Slider.Root origin="center" defaultValue={[0]} min={-100} max={100}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb index={0} aria-label="Pan" />
  </Slider.Track>
</Slider.Root>
```

## Styling hooks

The root and interactive parts expose `data-thumb-count`, `data-orientation`, `data-origin`, `data-inverted`, `--slider-percent`, and range variables.

```css
.slider-range {
  left: calc(var(--slider-range-start-percent) * 100%);
  width: calc(var(--slider-range-size-percent) * 100%);
}

.slider-thumb {
  left: calc(var(--slider-thumb-percent) * 100%);
}

[data-orientation="vertical"] .slider-range {
  bottom: calc(var(--slider-range-start-percent) * 100%);
  height: calc(var(--slider-range-size-percent) * 100%);
}
```

## API reference

### `Slider.Root`

`Slider.Root` renders a `div`, owns slider state, and provides context to the other parts. It accepts standard `div` props except `defaultValue`, `onChange`, and `children`, which are replaced by the slider-specific API.

| Prop                    | Type                                    | Default        | Description                                                                                                                              |
| ----------------------- | --------------------------------------- | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `value`                 | `number[]`                              | -              | Controlled thumb values. Pass one or more values.                                                                                        |
| `defaultValue`          | `number[]`                              | `[min]`        | Initial uncontrolled values. Also becomes the double-click reset target.                                                                 |
| `min`                   | `number`                                | `0`            | Lower value bound. Values are clamped into the resolved range.                                                                           |
| `max`                   | `number`                                | `100`          | Upper value bound. If `min` is greater than `max`, the core range resolver normalizes the order.                                         |
| `step`                  | `number`                                | `1`            | Value increment for pointer and keyboard changes. Non-positive or invalid steps fall back to the default.                                |
| `minStepsBetweenThumbs` | `number`                                | `0`            | Minimum number of `step` intervals between adjacent thumbs.                                                                              |
| `minDistance`           | `number`                                | `0`            | Minimum value-unit distance between adjacent thumbs. Ignored when `minStepsBetweenThumbs` is supplied.                                   |
| `thumbs`                | `{ min?: number; max?: number }[]`      | `[]`           | Optional per-thumb bounds, matched by index.                                                                                             |
| `orientation`           | `"horizontal" \| "vertical"`            | `"horizontal"` | Controls pointer mapping, `aria-orientation`, and `data-orientation`.                                                                    |
| `origin`                | `"left" \| "center" \| "right"`         | `"left"`       | Visual fill origin when there is one thumb. `center` is useful for pan controls; `right` fills back from the maximum edge.               |
| `inverted`              | `boolean`                               | `false`        | Reverses the value-to-position mapping and adds `data-inverted`.                                                                         |
| `disabled`              | `boolean`                               | `false`        | Prevents pointer and keyboard interaction, removes the thumb from tab order, and adds disabled state attributes.                         |
| `readOnly`              | `boolean`                               | `false`        | Keeps the thumb focusable while preventing value changes, and adds readonly state attributes.                                            |
| `fineControl`           | `boolean \| number`                     | `true`         | When `true`, Shift scales pointer drag movement to 10% for fine adjustment. Pass `false` to disable, or a number to set the drag factor. |
| `resetOnDoubleClick`    | `boolean`                               | `true`         | Allows a double click on the track to reset to `defaultValue`, or to `min`/`0` when no default value is supplied.                        |
| `allowTrackClick`       | `boolean`                               | `false`        | When true, pointer down on the track jumps to the pointer position before dragging. When false, dragging starts from the thumb only.     |
| `name`                  | `string`                                | -              | Default form field name consumed by `Slider.HiddenInput`.                                                                                |
| `required`              | `boolean`                               | -              | Default required flag consumed by `Slider.HiddenInput`.                                                                                  |
| `children`              | `ReactNode \| (state) => ReactNode`     | -              | Parts to render. Function children receive the render state, including value state plus interaction flags.                               |
| `onValueChange`         | `(value: number[]) => void`             | -              | Fires when interaction produces new clamped and snapped values.                                                                          |
| `onValueCommit`         | `(value: number[]) => void`             | -              | Fires when an interaction commits, such as pointer up, keyboard step, or reset.                                                          |
| `render`                | `RenderProp<ElementProps, SliderState>` | -              | Replaces the root element while preserving generated props and render state.                                                             |

### Parts

| Part                 | Props and behavior                                                                                                                                                                                                       |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Slider.Track`       | Renders a `div` and accepts standard `div` props plus `render`. It owns pointer drag handling, optional track click handling, nearest-thumb selection, and double-click reset.                                           |
| `Slider.Range`       | Renders an `aria-hidden` `span` and accepts standard `span` props plus `render`. Use it for the visual fill; it exposes `--slider-range-start-percent`, `--slider-range-end-percent`, and `--slider-range-size-percent`. |
| `Slider.Thumb`       | Renders a focusable `span` with `role="slider"` and generated ARIA value metadata. Pass `index` to bind it to the matching value.                                                                                        |
| `Slider.Value`       | Renders a `span` connected to the thumbs by `aria-describedby`. Without `index`, all values are joined with `-`; with `index`, it displays one thumb value.                                                              |
| `Slider.HiddenInput` | Renders `<input type="hidden">` for native form submission. Values are submitted as a JSON string such as `[20,80]`; `name`, `required`, and `disabled` default to root props.                                           |
