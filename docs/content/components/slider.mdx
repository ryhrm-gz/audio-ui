# Slider

`Slider` is a linear control for normalized parameters, pan, wet/dry balance, and other values that map cleanly to a percentage. It supports horizontal and vertical orientation.

<SliderDemo />

## Anatomy

```tsx
import { Slider } from "@ryhrm-gz/audio-ui-react";

export function PanSlider() {
  return (
    <Slider.Root defaultValue={0} min={-100} max={100} step={1} name="pan" origin="center">
      <Slider.Track>
        <Slider.Range />
        <Slider.Thumb aria-label="Pan" />
      </Slider.Track>
      <Slider.Value format={(value) => `${value}%`} />
      <Slider.HiddenInput />
    </Slider.Root>
  );
}
```

## Vertical orientation

```tsx
<Slider.Root orientation="vertical" defaultValue={50} min={0} max={100}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb aria-label="Send level" />
  </Slider.Track>
</Slider.Root>
```

## Range origin

Use `origin` to choose where the visual range starts. The default is `left`; `center` works well for pan controls, and `right` fills back from the end of the track.

```tsx
<Slider.Root origin="center" defaultValue={0} min={-100} max={100}>
  <Slider.Track>
    <Slider.Range />
    <Slider.Thumb aria-label="Pan" />
  </Slider.Track>
</Slider.Root>
```

## Styling hooks

The root and interactive parts expose `data-orientation`, `data-origin`, `data-inverted`, `--slider-percent`, and range-origin variables.

```css
.slider-range {
  left: calc(var(--slider-range-start-percent) * 100%);
  width: calc(var(--slider-range-size-percent) * 100%);
}

[data-orientation="vertical"] .slider-range {
  bottom: calc(var(--slider-range-start-percent) * 100%);
  height: calc(var(--slider-range-size-percent) * 100%);
}
```

## API reference

### `Slider.Root`

`Slider.Root` renders a `div`, owns slider state, and provides context to the other parts. It accepts standard `div` props except `defaultValue`, `onChange`, and `children`, which are replaced by the slider-specific API.

| Prop                 | Type                                    | Default        | Description                                                                                                                          |
| -------------------- | --------------------------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `value`              | `number`                                | -              | Controlled value. Pass with `onValueChange` when external state owns the slider value.                                               |
| `defaultValue`       | `number`                                | `min` or `0`   | Initial uncontrolled value. Also becomes the double-click reset target.                                                              |
| `min`                | `number`                                | `0`            | Lower value bound. Values are clamped into the resolved range.                                                                       |
| `max`                | `number`                                | `100`          | Upper value bound. If `min` is greater than `max`, the core range resolver normalizes the order.                                     |
| `step`               | `number`                                | `1`            | Value increment for pointer and keyboard changes. Non-positive or invalid steps fall back to the default.                            |
| `orientation`        | `"horizontal" \| "vertical"`            | `"horizontal"` | Controls pointer mapping, `aria-orientation`, and `data-orientation`.                                                                |
| `origin`             | `"left" \| "center" \| "right"`         | `"left"`       | Sets the visual fill origin. `center` is useful for pan controls; `right` fills back from the maximum edge.                          |
| `inverted`           | `boolean`                               | `false`        | Reverses the value-to-position mapping and adds `data-inverted`.                                                                     |
| `disabled`           | `boolean`                               | `false`        | Prevents pointer and keyboard interaction, removes the thumb from tab order, and adds disabled state attributes.                     |
| `readOnly`           | `boolean`                               | `false`        | Keeps the thumb focusable while preventing value changes, and adds readonly state attributes.                                        |
| `fineControl`        | `boolean \| number`                     | `true`         | When `true`, Shift uses 10% of `step` for fine adjustment. Pass `false` to disable, or a number to set the fine step explicitly.     |
| `resetOnDoubleClick` | `boolean`                               | `true`         | Allows a double click on the track to reset to `defaultValue`, or to `min`/`0` when no default value is supplied.                    |
| `allowTrackClick`    | `boolean`                               | `false`        | When true, pointer down on the track jumps to the pointer position before dragging. When false, dragging starts from the thumb only. |
| `name`               | `string`                                | -              | Default form field name consumed by `Slider.HiddenInput`.                                                                            |
| `required`           | `boolean`                               | -              | Default required flag consumed by `Slider.HiddenInput`.                                                                              |
| `children`           | `ReactNode \| (state) => ReactNode`     | -              | Parts to render. Function children receive the render state, including value state plus interaction flags.                           |
| `onValueChange`      | `(value: number) => void`               | -              | Fires when interaction produces a new clamped and snapped value.                                                                     |
| `onValueCommit`      | `(value: number) => void`               | -              | Fires when an interaction commits, such as pointer up, keyboard step, or reset.                                                      |
| `render`             | `RenderProp<ElementProps, SliderState>` | -              | Replaces the root element while preserving generated props and render state.                                                         |

### Parts

| Part                 | Props and behavior                                                                                                                                                                                                                                                               |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Slider.Track`       | Renders a `div` and accepts standard `div` props plus `render`. It owns pointer drag handling, optional track click handling, and double-click reset. It exposes `data-part="track"`, orientation/origin/inverted state attributes, dragging state, and slider CSS variables.    |
| `Slider.Range`       | Renders an `aria-hidden` `span` and accepts standard `span` props plus `render`. Use it for the visual fill; it exposes `--slider-range-start-percent`, `--slider-range-end-percent`, and `--slider-range-size-percent` so CSS can position the fill from the selected `origin`. |
| `Slider.Thumb`       | Renders a focusable `span` with `role="slider"` and generated ARIA value metadata. It accepts standard `span` props except `defaultValue` and `onChange`, plus `render`. Arrow keys, Page Up/Down, Home, and End are handled by the generated keyboard behavior.                 |
| `Slider.Value`       | Renders a `span` connected to the thumb by `aria-describedby`. It accepts standard `span` props except `children`, plus `children`, `format`, and `render`. `format(value, state)` customizes display text when explicit children are not supplied.                              |
| `Slider.HiddenInput` | Renders `<input type="hidden">` for native form submission. It accepts input props except `children`, `type`, and `value`; `name`, `required`, and `disabled` default to the matching root props and can be overridden directly on the hidden input.                             |
